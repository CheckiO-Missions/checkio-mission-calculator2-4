Першими відомими інструментами, що використовувалися для полегшення арифметичних підрахунків, були: кістки, галька та абакус. Відомо, що їх використовували шумери та єгиптяни до 2000 р. до н.е. За винятком Антикітерського механізму (астрономічного пристрою, що значно випереджав свій час), розробка обчислювальних інструментів розпочалася близько початку 17 століття: геометричний військовий компас (Галілей), логарифми та кістки Нап'єра та логарифмічна лінійка (Едмунд Гюнтер). У цій серії місій ти маєш створити елементарний калькулятор.

В якості вхідних даних, ти отримаєш послідовність натискань клавіш, і, як результат твоєї функції, повинні повернути, що буде показано на екрані після натискання останньої клавіші. Будь уважний, те, що буде показано, не завжди є результатом виразу. Власне, гра з фізичним калькулятором або додатком дійсно допоможе тобі зрозуміти крайові варіанти.

У четвертій місії додаються числа з рухомою крапкою (float). "." займає місце однієї цифри, тож максимальна кількість цифр для такого числа - 4. Це означає, що найбільше число з рухомою крапкою, яке можна ввести - "9999.".

Очікувана поведінка:

  • початкові операції ігноруються;
  • початкові нулі мають бути забрані, число лише з нулів - замінене на один нуль;
  • серед +- знаків між числами слід брати останній;
  • "==" означає повторення останньої операції;
  • "+=" або "-=" - додавання/віднімання числа (або результату операцій) перед комбінацією (подвоєння числа/віднімання самого від себе);
  • калькулятор ігнорує цифри, які вводяться, після 5-ї;
  • "-" для чисел < 0 НЕ займає місця цифри;
  • якщо абс.результат більше 99999 - "error" показується як результат;
  • для float, якщо цілочисельна частина абс.результату більша 9999 - "error" показується як результат;
  • для float, якщо загальна довжина числа більше 5 цифр, воно повинно бути заокруглене до 5 цифр (1.23456 -> 1.234);
  • для float, початкові і кінцеві нулі мають бути вилучені (навіть до ".", якщо можливо): 0.1200 -> .12 , 123.00 -> 123. . Це має бути зроблене після заокруглення: 1.000123 -> 1. . Вилучення кінцевих нулів повинно робитися лише після завершення введення числа (введено не цифру).
