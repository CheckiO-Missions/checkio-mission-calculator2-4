Les premiers outils connus utilisés pour faciliter les calculs arithmétiques sont les os (utilisés pour compter les objets), les cailloux, les tableaux de comptage et l'abaque, dont on sait qu'il a été utilisé par les Sumériens et les Égyptiens avant 2000 avant Jésus-Christ. À l'exception du mécanisme d'Antikythera (un dispositif astronomique "hors du temps"), les outils de calcul ont été mis au point vers le début du XVIIe siècle: le compas géométrique militaire (par Galilée), les logarithmes et les os de Napier (par Napier) et la règle à calcul (par Edmund Gunter). Dans cette série de missions, vous allez construire une calculatrice élémentaire.

En entrée, vous recevez une séquence de touches enfoncées et, en résultat de cette fonction, vous devez montrer ce qui sera affiché à l'écran lorsque la dernière touche sera enfoncée. Attention, ce n'est pas toujours le résultat de l'expression. En fait, jouer avec la calculatrice physique ou l'application vous aidera vraiment à repérer les cas limites.

Dans la quatrième mission, les nombres flottants sont autorisés. Le "." prend l'espace d'un chiffre, donc l'espace maximum est de 4 pour les résultats des nombres flottants. Cela signifie que le nombre flottant maximum que vous pouvez saisir est "9999".

Comportement attendu:

  • les zéros du début doivent être supprimés, les nombres à zéros uniques - convertis en zéro unique;
  • parmi les signes +- entre les nombres, le dernier doit être pris;
  • "==" signifie que l'on répète la dernière opération;
  • "+=" ou "-=" - ajouter/soustraire le nombre (ou le résultat de l'opération) avant la combinaison (doubler le nombre/soustraire lui-même);
  • la calculatrice ignore les chiffres que vous entrez après le 5;
  • "-" pour les nombres < 0 ne prend PAS la place d'un chiffre;
  • si l'abs(résultat) est supérieur à 99999 - "erreur" est affiché comme résultat;
  • pour les flottants, si la partie entière de abs(result) est supérieure à 9999 - "error" est affiché comme résultat;
  • pour les flottants, si la longueur totale du nombre est supérieure à 5 chiffres, elle doit être arrondie à 5 chiffres (1.23456 -> 1.235);
  • pour les flottants, les zéros de début et de fin doivent être supprimés (jusqu'au "." si possible): 0.1200 -> .12 , 123.00 -> 123. . Elle doit être effectuée après l'arrondi: 1.000123 -> 1. . L'élimination des zéros de fin ne doit être effectuée qu'après la fin de la saisie d'un nombre (appui sur un caractère autre qu'un chiffre).
